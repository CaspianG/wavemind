# Support

Use GitHub Issues for reproducible bugs, benchmark requests, and integration
requests. The repository includes issue templates for those paths.

Before opening an issue:

- check `README.md` and `docs/ROADMAP.md`;
- include the exact command or code snippet;
- include Python version, OS, installed extras, and WaveMind version;
- include benchmark JSON when discussing benchmark results.

For general roadmap discussion, use GitHub Discussions when enabled for the
repository. Until a public chat is created, GitHub is the source of truth for
support and project direction.
